# 🐛 Как отладить проблему с удалением из корзины

## Быстрая инструкция

### Метод 1: Проверка в основном приложении

1. Откройте https://enterprises-database.web.app/
2. Нажмите **F12** (или **Ctrl+Shift+I**) — откроется DevTools
3. Перейдите на вкладку **Console**
4. Откройте **Корзину** (кнопка вверху)
5. Нажмите **"Удалить навсегда"** на любом предприятии
6. Смотрите логи в консоли:

**Что искать:**
```
✅ Функция window.deletePermanently определена и доступна
🗑️ ========== deletePermanently ВЫЗВАНА ==========
🎯 Входной параметр trashId: XYZ123...
```

Если эти логи НЕ появляются — проблема в кнопке (onclick не работает).
Если появляются — смотрите дальше, где именно падает ошибка.

### Метод 2: Изолированный тест

1. Откройте https://enterprises-database.web.app/test-trash-delete.html
2. Нажмите **"Загрузить корзину"**
3. Нажмите **"Удалить навсегда"** на любом элементе
4. Смотрите логи прямо на странице (не нужен DevTools!)

### Метод 3: Тест прав доступа Firestore

1. Откройте https://enterprises-database.web.app/test-firestore-permissions.html
2. Последовательно нажимайте кнопки:
   - **"Тест чтения"** — должен показать ✅
   - **"Тест добавления"** — должен показать ✅
   - **"Тест удаления"** — должен показать ✅
   - **"Полный тест"** — выполнит весь цикл

Если хотя бы один тест падает — проблема в правах Firestore.

## Типичные причины

### ❌ Кнопка вообще не реагирует
**Причина:** Старая версия сайта в кэше браузера
**Решение:** 
- Нажмите **Ctrl+Shift+R** (жёсткая перезагрузка)
- Или очистите кэш браузера

### ❌ Ошибка "permission-denied"
**Причина:** Проблема с правами доступа Firestore
**Решение:** Проверьте firestore.rules — должно быть `allow read, write: if true;`

### ❌ Ошибка "not-found"
**Причина:** Документ уже удалён или ID неправильный
**Решение:** Перезагрузите корзину

### ❌ Функция вызывается, но ничего не происходит
**Причина:** Сетевая ошибка или проблема с Firebase
**Решение:** Проверьте интернет-соединение, проверьте статус Firebase

## Дополнительные шаги

### Если ничего не помогает:

1. **Проверьте консоль на предмет ошибок JavaScript**
   - Красные сообщения в Console
   - Возможно, другая ошибка ломает весь скрипт

2. **Проверьте Network во вкладке DevTools**
   - Фильтруйте по `firestore`
   - Смотрите, идут ли запросы и какой статус

3. **Проверьте вкладку Application → Console (если есть Service Worker)**
   - Возможно, Service Worker кэширует старую версию

4. **Попробуйте в режиме инкогнито**
   - Это исключит проблемы с кэшем и расширениями

## Отправка отчёта

Если проблема продолжается, пришлите:
1. **Скриншот консоли** (вкладка Console в DevTools)
2. **Скриншот Network** (вкладка Network, фильтр firestore)
3. **Результат теста** из test-trash-delete.html

Это поможет точно определить причину!
