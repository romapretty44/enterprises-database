# Руководство по тестированию базы предприятий

## Реализованные функции

### 1. ✅ Динамические фильтры (отрасли)
**Как протестировать:**
1. Откройте модальное окно добавления предприятия
2. Внизу списка отраслей в фильтрах нажмите "➕ Добавить отрасль"
3. Введите название новой отрасли (например, "IT-сектор")
4. Новая отрасль появится в списке фильтров и в форме добавления

### 2. ✅ Загрузка файлов
**Как протестировать:**
1. Откройте модальное окно добавления/редактирования предприятия
2. В разделе "📎 Файлы" нажмите "Выбрать файлы"
3. Выберите документы или изображения
4. Сохраните предприятие
5. Откройте карточку на просмотр - файлы будут доступны для скачивания

**Поддерживаемые типы:** любые файлы (документы, изображения, PDF и т.д.)

### 3. ✅ Экспорт базы в Excel/CSV
**Как протестировать:**
1. Нажмите кнопку "📊 Экспорт в Excel" в шапке
2. Скачается CSV-файл с названием `enterprises_YYYY-MM-DD.csv`
3. Откройте в Excel (формат: Данные → Из текста/CSV → разделитель точка с запятой)
4. Проверьте, что все данные экспортированы корректно

**Экспортируются:**
- Название
- Информация
- Отрасли
- Категории
- Поддержка региональная/федеральная
- Контакты

### 4. ✅ Корзина удалённых
**Как протестировать:**
1. Удалите любое предприятие (кнопка "🗑️ Удалить")
2. Нажмите кнопку "🗑️ Корзина" в шапке
3. Предприятие будет в корзине с датой удаления
4. Нажмите "♻️ Восстановить" для восстановления
5. Или "🗑️ Удалить навсегда" для окончательного удаления

### 5. ✅ Компактные карточки
**Как протестировать:**
1. На главной странице карточки показываются БЕЗ описания
2. Отображается только название и отрасли
3. Нажмите "👁️ Просмотр" для открытия модального окна
4. В модальном окне показывается вся информация, включая описание

### 6. ✅ Фильтрация AND (вместо OR)
**Как протестировать:**
1. Создайте предприятие с отраслями "Промышленность" и "ОПК"
2. Создайте предприятие с отраслью только "Промышленность"
3. Выберите в фильтрах "Промышленность" и "ОПК"
4. Должно показаться только первое предприятие (у которого ОБЕ отрасли)

**Логика:** предприятие должно содержать ВСЕ выбранные фильтры, а не хотя бы один

### 7. ✅ Дополнительные категории
**Как протестировать:**
1. Откройте форму добавления/редактирования
2. В разделе "📂 Дополнительные категории" установите чекбоксы:
   - Категорируется
   - Участник производительности труда
   - Участник профессионалитета
   - Получатель поддержки региональной (+ текстовое поле)
   - Получатель поддержки федеральной (+ текстовое поле)
3. Сохраните предприятие
4. В модальном окне просмотра отобразятся выбранные категории
5. В фильтрах можно фильтровать по этим категориям

### 8. ✅ Раздел контактов
**Как протестировать:**
1. Откройте форму добавления/редактирования
2. В разделе "👥 Контакты" нажмите "➕ Добавить контакт"
3. Заполните поля:
   - ФИО
   - Должность
   - Рабочий телефон
   - Почта
   - Мобильный телефон
4. Добавьте несколько контактов (неограниченное количество)
5. Удалите ненужные контакты кнопкой "❌"
6. Сохраните и проверьте в модальном окне просмотра

## Проверка безопасности

### Firestore Rules
Текущие правила: открытый доступ для чтения/записи.
В будущем рекомендуется добавить Firebase Authentication.

### Storage Rules
Текущие правила: открытый доступ для загрузки/скачивания файлов в папке `/enterprises/`.

## Деплой Firebase правил

```bash
# Деплой Firestore правил
firebase deploy --only firestore:rules

# Деплой Storage правил
firebase deploy --only storage:rules

# Деплой всего
firebase deploy
```

## Известные ограничения

1. **Авторизация:** жёсткий пароль в коде. Рекомендуется внедрить Firebase Auth.
2. **Размер файлов:** максимум 50MB по умолчанию (ограничение Firebase Storage).
3. **CSV экспорт:** использует точку с запятой как разделитель (для Excel).
4. **Real-time синхронизация:** работает через Firestore listeners, все изменения видны сразу.
